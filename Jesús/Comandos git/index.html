INICIANDO CON GIT

Git es un control de versiones que permite a equipos de desarrolladores trabajar de manera remota en un mismo proyecto. Permite hacer contribuciones de código de manera local y luego replicarlos a un repositorio remoto.


Principales comandos con Git

  • pwd: sirve para saber en qué directorio estamos.
  • ls: enlista los directorios existentes.
  • cd: se usa para volver al directorio raíz del usuario.
  • cd ..: regresa dos directorios respecto al actual.
  • mkdir "nombre_carpeta" crea un directorio/carpeta nuevo.
  • Git init: funciona para inicializar un proyecto en git.
  • Git clone "url_repositorio": sirve para clonar un repositorio público.
  • Git branch: muestra la rama del repositorio local.
  • Git branch -M "sombre_rama" renombra la rama actual.
  • Git checkout -b "nombre_rama": cambia y crea otra rama.
  • Git checkeout -- .: es como un Ctrl + Z. Sirve para deshacer los cambios no confirmados.
  • Git config user.email: muestra el email agregado.
  • Git config user.name: muestra el nombre de usuario agregado.
  • Git config --global user.email "email": sirve para identificar añadir globalmente un email a la terminal.
  • Git config --global user.name "name": se emplea para identificar y configurar globalmente el nombre del usuario con el repositorio virtual.
  • Git status: permite verificar el estado del repositorio local.
  • Git add .: se utiliza para agregar todos los archivos modificados y nuevos en el repositorio local.
  • Git commit -m "comentario": permite guardar los cambios realizados en el repositorio local con un mensaje resumido de los cambios.
  • Git push: sirve para enviar los cambios confirmados a un repositorio existente.
  • Git remote add origin "link_repositorio": envía el repositorio local a un repositorio remoto como GitHub.
  • Git push -u origin main: funciona para enviar el repositorio local al repositorio remoto de GitHub.
  • Git log: muestra el historial de cambios en el repositorio local.
